Integration Reference

Auth methods, retry policy, and key behaviours per service

External

NetSuite REST API

  • Auth: OAuth 1.0a TBA, HMAC-SHA256 signed per request
  • createVendor on submission
  • createTask → Finance (submission)
  • updateVendorStatus → NDA Complete
  • createTask → Legal (NDA signed)
  • Retry: up to 5× exponential backoff
External

Acrobat Sign

  • Auth: OAuth 2.0 refresh token, cached + auto-refreshed 60s before expiry
  • sendNdaAgreement on submission
  • downloadSignedNda on completion
  • getAgreementStatus (polling fallback)
  • Webhook: client ID echo only
  • Polling: every 5 min
External

SendGrid

  • Auth: API key (Bearer token)
  • sendInternalAlert on submission
  • sendWelcomeEmail on NDA complete
  • Welcome email includes signed NDA + authorization letter as attachments
  • Retry: up to 5× exponential backoff
AWS

AWS S3

  • Auth: IAM credentials via env vars
  • All files: AES-256 SSE, private ACL
  • w9.{ext}, bank_letter.{ext} — on submission
  • vendor_setup_form.pdf — worker
  • signed_nda.pdf — after NDA complete
  • Key pattern: resellers/{id}/
AWS

AWS SQS

  • Auth: IAM credentials
  • Job types: RESELLER_SUBMITTED, NDA_COMPLETED
  • Worker long-polls continuously
  • Messages acked after max retries to prevent infinite loops
  • Recommended: configure DLQ in production
Database

PostgreSQL

  • Primary store for reseller records
  • EIN unique key — resubmissions upsert
  • Tracks status: Initiated → NDA Pending → NDA Complete → Cancelled
  • Stores S3 keys, NetSuite vendor ID, envelope ID
Internal

PDF Generator

  • PDFKit: programmatic PDF generation
  • Carbone: template-based generation
  • generateVendorSetupForm — on submission
  • generateAuthorizationLetter — on NDA complete
API

Security & Rate Limiting

  • Helmet: security headers
  • Global rate limiter on all routes
  • Submission rate limit: 5 req/hr per IP
  • JWT (jsonwebtoken) — auth tokens
  • Multer + multer-s3: streams uploads direct to S3 (no disk temp files)
